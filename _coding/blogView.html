Github 블로그 만드는 방법

웹쓰 2023.04.03
수정 삭제

GitHub Actions를 사용하여 자동화된 워크플로우를 설정하는 방법을 단계별로 안내하겠습니다. GitHub Actions는 소프트웨어 개발 워크플로를 자동화할 수 있게 해주며, 특히 CI/CD (Continuous Integration/Continuous Deployment) 작업에 유용합니다.

1. GitHub 저장소 준비 : GitHub Actions를 사용하기 위해 GitHub에 저장소가 필요합니다. 저장소가 없다면 새로 생성해야 합니다.

2. Actions 탭 활성화 : GitHub 저장소에서 'Actions' 탭을 클릭하여 GitHub Actions 화면으로 이동합니다.

3. 워크플로우 파일 생성 워크플로우는 .github/workflows 디렉토리에 YAML 파일 형식으로 저장됩니다. 워크플로우를 처음 설정할 때 GitHub은 일반적인 사용 사례에 맞는 워크플로우 템플릿을 제안할 수 있습니다. 이 파일은 push와 pull_request 이벤트에 반응하여 Ubuntu 가상 환경에서 세 단계의 작업을 실행합니다. 첫 번째 단계는 저장소 코드를 체크아웃하는 것이고, 이후 두 단계에서는 간단한 쉘 스크립트를 실행합니다.
